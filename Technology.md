# AI File Searcher - 技術仕様書

このドキュメントでは、AI File Searcherの内部構造、アーキテクチャ、およびセットアップに関する詳細な技術情報を解説します。

## ⚙️ システム構成

本ツールは、Node.jsによる軽量なローカルサーバーと、GoogleのGemini APIを統合した構成になっています。

### アーキテクチャ
- **Backend**: Node.js (Express)
  - ファイルシステムの走査、AI APIとの通信、サジェストの生成を担当。
- **Frontend**: Vanilla HTML/CSS/JS
  - プレミアムなダークモードUI、リアルタイムな進捗表示、拡張子ピッカーを制御。
- **OS連携**: PowerShell (Win32 API P/Invoke)
  - Windows標準のフォルダ選択ダイアログを、OSのフォーカス制限を突破して最前面に表示。

## 🚀 技術的な特徴

### 1. 非同期ストリーミング・スキャン
ファイル走査処理は `fs.promises` を利用した非同期再帰アルゴリズムで実装されています。各検索には固有の **Search ID** が割り当てられ、スキャン中もイベントループをブロックせずにフロントエンドからの進捗確認（ポーリング）やキャンセル要求を受け付けることが可能です。

### 2. コンテキスト指向プロンプト
AIへのプロンプトには、単なるファイルパスだけでなく以下の情報が動的に組み込まれます：
- **現在の日時**: 「昨日」「先週」などの相対的な時間表現を解釈可能にするため。
- **ファイルの更新日時**: `fs.stat` から取得したタイムスタンプ。
- **学習履歴（Open Count）**: `history.json` から取得した、過去にユーザーがそのファイルを開いた回数。
- **内容の断片**: 特定の拡張子（.txt, .md等）において、トークン制限内で先頭数百文字を抽出。

### 3. 動的クエリ・インジェクション
フロントエンドで検索キーワードが未入力かつ拡張子が選択されている場合、システムは自動的に「指定された拡張子のファイルをリストアップしてください」という指示を生成し、AIに渡します。これにより、AIの推論能力を活かした拡張子フィルタリングが可能になっています。

### 4. AI学習機能（パーソナライズ）
ユーザーが検索結果をクリックするたびに、`/api/open` エンドポイントを介して `history.json` にそのパスの「開封回数」がインクリメントされます。次回の検索プロンプトにおいて、この回数がAIに伝えられ、「よく使われるファイル（お気に入り）」として関連度（relevance）を意図的に高く評価させる（1.5倍ブースト）仕組みを導入しています。

### 3. Windowsネイティブ連携
`browse.ps1` は、PowerShellから直接Windows Formsを呼び出すだけでなく、C#コードを動的にコンパイルして `SetForegroundWindow` 等のWin32 APIを呼び出します。これにより、ブラウザの裏にダイアログが隠れる問題（Foreground Lock）を解決しています。

## 🛠 セットアップ・導入手順

### 必要な環境
- Node.js (v16.x 以上)
- Google AI (Gemini) APIキー
- OS: Windows 10/11 (推奨)

### インストール
1. リポジトリをクローンまたはダウンロードします。
2. ルートディレクトリで `npm install` を実行し、依存パッケージをインストールします。

### 設定
`apis.json` を開き、APIキーを設定します。
```json
{
  "apiKey": "YOUR_API_KEY",
  "models": ["gemini-3.1-flash-lite-preview"]
}
```

### 実行
`run.bat` を実行すると、サーバーが起動しブラウザが自動的に開きます。
